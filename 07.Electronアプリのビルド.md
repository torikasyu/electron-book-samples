# 第7章: Electronアプリのビルド

この章では、Electron + React + TypeScriptで開発したアプリケーションをビルドし、配布可能なパッケージに変換する基本的な方法について解説します。初心者の方でも理解しやすいよう、シンプルな内容にしています。

## ビルドの基本設定

Electronアプリケーションは、主にWindows、macOSの2つのプラットフォームで動作させることができます。ここでは、これらのプラットフォーム向けにビルドするための基本的な設定方法を解説します。

### electron-builderの導入

Electronアプリケーションのビルドには、`electron-builder`というツールが広く使用されています。以下のコマンドでインストールします：

```bash
npm install --save-dev electron-builder
```

### package.jsonの設定

`package.json`ファイルに、ビルド設定を追加します：

```json
{
  "name": "electron-image-app",
  "version": "1.0.0",
  "description": "画像管理アプリケーション",
  "main": "dist/main.js",
  "scripts": {
    "start": "electron .",
    "build": "webpack --config webpack.config.js",
    "dev": "concurrently \"npm run build -- --watch\" \"npm run start\"",
    "package": "npm run build && electron-builder build --publish never",
    "package:win": "npm run build && electron-builder build --win --publish never",
    "package:mac": "npm run build && electron-builder build --mac --publish never"
  },
  "build": {
    "appId": "com.example.electron-image-app",
    "productName": "画像管理アプリ",
    "files": [
      "dist/**/*",
      "node_modules/**/*",
      "package.json"
    ],
    "directories": {
      "buildResources": "resources",
      "output": "release"
    },
    "mac": {
      "category": "public.app-category.utilities",
      "target": ["dmg", "zip"]
    },
    "win": {
      "target": ["nsis", "portable"]
    },
    "publish": {
      "provider": "github",
      "owner": "your-github-username",
      "repo": "your-repo-name"
    }
  }
}
```

### アイコンとリソースの準備

アプリケーションのアイコンなどのリソースを準備します。以下のようなディレクトリ構造を作成します：

```
resources/
├── icon.icns      # macOS用アイコン
└── icon.ico       # Windows用アイコン
```

各プラットフォーム用のアイコンは、以下の仕様に合わせて作成します：

- **Windows (icon.ico)**: 256x256ピクセル
- **macOS (icon.icns)**: 1024x1024ピクセル

### プラットフォーム固有の設定

#### Windows向けの設定

Windows向けのビルド設定を詳細に指定するには、`package.json`の`build`セクションに以下のように追加します：

```json
"win": {
  "target": ["nsis", "portable"],
  "icon": "resources/icon.ico",
  "artifactName": "${productName}-${version}-setup.${ext}",
  "publisherName": "あなたの会社名"
},
"nsis": {
  "oneClick": false,
  "allowToChangeInstallationDirectory": true,
  "createDesktopShortcut": true,
  "createStartMenuShortcut": true,
  "shortcutName": "画像管理アプリ"
}
```

#### macOS向けの設定

macOS向けのビルド設定を詳細に指定するには、以下のように追加します：

```json
"mac": {
  "category": "public.app-category.utilities",
  "target": ["dmg", "zip"],
  "icon": "resources/icon.icns",
  "hardenedRuntime": true,
  "gatekeeperAssess": false,
  "entitlements": "build/entitlements.mac.plist",
  "entitlementsInherit": "build/entitlements.mac.plist"
},
"dmg": {
  "contents": [
    {
      "x": 130,
      "y": 220
    },
    {
      "x": 410,
      "y": 220,
      "type": "link",
      "path": "/Applications"
    }
  ],
  "window": {
    "width": 540,
    "height": 380
  }
}
```

macOSのコード署名とノータリゼーションを行うには、追加の設定が必要です。詳細は後述します。



## パッケージング

### ビルドプロセスの実行

アプリケーションをパッケージ化するには、以下のコマンドを実行します：

```bash
# すべてのプラットフォーム向けにビルド
npm run package

# Windows向けにビルド
npm run package:win

# macOS向けにビルド
npm run package:mac
```

ビルドが完了すると、`release`ディレクトリに配布可能なパッケージが生成されます。

### クロスプラットフォームビルドの注意点

異なるプラットフォーム向けにビルドする場合、以下の点に注意が必要です：

1. **Windows向けのビルド**:
   - macOSからWindows向けにビルドする場合、Wine（Windowsエミュレーター）が必要です。
   - `brew install --cask wine-stable`でインストールできます。

2. **macOS向けのビルド**:
   - macOS向けのビルドは、macOS上でのみ可能です。
   - コード署名とノータリゼーションには、Apple Developer Programへの登録が必要です。

### コード署名とノータリゼーション（macOS）

macOSでは、Gatekeeperという機能により、署名されていないアプリケーションの実行が制限されています。本格的に配布する場合は、コード署名とノータリゼーションが必要になります。

#### 1. Apple Developer Programに登録

Apple Developer Program（年間99ドル）に登録し、開発者証明書を取得します。

#### 2. 証明書の取得

Xcode > Preferences > Accounts から、Apple IDを追加し、「Manage Certificates」をクリックして証明書を作成します。

#### 3. ビルドスクリプトの修正

`package.json`の`scripts`セクションに、コード署名用のスクリプトを追加します：

```json
"scripts": {
  "package:mac:signed": "npm run build && electron-builder build --mac --publish never -c.mac.identity='Developer ID Application: あなたの名前 (チームID)'"
}
```

※ 個人で使用する場合や開発段階では、コード署名なしでも問題なく動作させることができます。

### Windows用インストーラーのカスタマイズ

Windows用のインストーラー（NSIS）をカスタマイズするには、以下のような設定を追加します：

```json
"nsis": {
  "oneClick": false,
  "allowToChangeInstallationDirectory": true,
  "createDesktopShortcut": true,
  "createStartMenuShortcut": true,
  "shortcutName": "画像管理アプリ",
  "installerIcon": "resources/icon.ico",
  "uninstallerIcon": "resources/icon.ico",
  "installerHeaderIcon": "resources/icon.ico",
  "installerSidebar": "resources/installer-sidebar.bmp",
  "uninstallerSidebar": "resources/installer-sidebar.bmp",
  "license": "LICENSE.txt"
}
```

サイドバー画像（`installer-sidebar.bmp`）は、164x314ピクセルのBMP形式で作成します。

## 自動アップデート機能の実装

Electronアプリケーションに自動アップデート機能を実装する方法を解説します。

### electron-updaterのインストール

```bash
npm install electron-updater
```

### 自動アップデート機能の実装

`src/main/main.ts`ファイルに、自動アップデート機能を実装します：

```typescript
import { app, BrowserWindow } from 'electron';
import * as path from 'path';
import { autoUpdater } from 'electron-updater';

let mainWindow: BrowserWindow | null = null;

function createWindow() {
  mainWindow = new BrowserWindow({
    width: 1200,
    height: 800,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js'),
      contextIsolation: true,
      nodeIntegration: false,
    },
  });

  mainWindow.loadFile(path.join(__dirname, 'index.html'));

  // 開発環境では開発者ツールを開く
  if (process.env.NODE_ENV === 'development') {
    mainWindow.webContents.openDevTools();
  }

  mainWindow.on('closed', () => {
    mainWindow = null;
  });

  // 自動アップデートの基本的なイベントハンドラー
  autoUpdater.on('update-available', () => {
    sendStatusToWindow('新しいバージョンが利用可能です');
  });

  autoUpdater.on('update-downloaded', () => {
    sendStatusToWindow('アップデートがダウンロードされました。再起動します');
    // 3秒後に再起動
    setTimeout(() => {
      autoUpdater.quitAndInstall();
    }, 3000);
  });

  // アプリケーション起動時にアップデートをチェック
  autoUpdater.checkForUpdatesAndNotify();
}

// メインウィンドウにステータスメッセージを送信
function sendStatusToWindow(text: string) {
  if (mainWindow) {
    mainWindow.webContents.send('update-message', text);
  }
}

app.whenReady().then(() => {
  createWindow();

  app.on('activate', () => {
    if (BrowserWindow.getAllWindows().length === 0) {
      createWindow();
    }
  });
});

app.on('window-all-closed', () => {
  if (process.platform !== 'darwin') {
    app.quit();
  }
});
```

### プリロードスクリプトの更新

`src/main/preload.ts`ファイルを更新して、アップデートメッセージをレンダラープロセスに渡せるようにします：

```typescript
import { contextBridge, ipcRenderer } from 'electron';

contextBridge.exposeInMainWorld('electron', {
  // 既存のAPI
  // ...

  // アップデート関連のAPI
  onUpdateMessage: (callback: (message: string) => void) => {
    ipcRenderer.on('update-message', (_event, message) => callback(message));
    return () => {
      ipcRenderer.removeAllListeners('update-message');
    };
  }
});
```

### レンダラープロセスでのアップデート通知の表示

`src/renderer/App.tsx`ファイルに、アップデート通知を表示する機能を追加します：

```tsx
import React, { useState, useEffect } from 'react';
// 他のインポート

const App: React.FC = () => {
  // 既存の状態
  // ...

  // アップデートメッセージの状態
  const [updateMessage, setUpdateMessage] = useState<string | null>(null);

  // アップデートメッセージのリスナーを設定
  useEffect(() => {
    // windowオブジェクトの拡張
    const electron = (window as any).electron;

    if (electron && electron.onUpdateMessage) {
      const removeListener = electron.onUpdateMessage((message: string) => {
        setUpdateMessage(message);
        // 5秒後にメッセージを消す
        setTimeout(() => setUpdateMessage(null), 5000);
      });

      return removeListener;
    }
  }, []);

  return (
    <div className="container mx-auto p-4">
      {/* 既存のコンポーネント */}
      {/* ... */}

      {/* アップデート通知 */}
      {updateMessage && (
        <div className="fixed bottom-4 right-4 bg-blue-500 text-white p-4 rounded-lg shadow-lg">
          {updateMessage}
        </div>
      )}
    </div>
  );
};

export default App;
```

### GitHub Releasesを使った配布

自動アップデート機能を使用するには、アプリケーションの新しいバージョンをGitHub Releasesなどで配布する必要があります。

#### 1. package.jsonの設定

`package.json`の`build`セクションに、以下の設定を追加します：

```json
"publish": {
  "provider": "github",
  "owner": "your-github-username",
  "repo": "your-repo-name",
  "private": false,
  "releaseType": "release"
}
```

#### 2. GitHubトークンの設定

GitHub Personal Access Tokenを生成し、環境変数として設定します：

```bash
export GH_TOKEN=your-github-token
```

#### 3. リリースの作成

新しいバージョンをリリースするには、以下の手順を実行します：

1. `package.json`のバージョンを更新します（例: `"version": "1.0.1"`）。
2. 以下のコマンドを実行してビルドとリリースを行います：

```bash
npm run build && electron-builder build --publish always
```

これにより、GitHubリリースページに新しいバージョンが公開され、ユーザーのアプリケーションは次回起動時に自動的にアップデートを検出します。
